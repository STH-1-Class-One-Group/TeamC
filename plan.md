### 프로젝트

대표 주제 : 군대 음식 기반의 웹사이트

사용 언어 및 스택 : 파이썬(fastapi - 백엔드)/자바스크립트(리액트 - 프론트엔드)/postgreSQL(데이터베이스)/cloudflare ⇒ AWS(좀 더 규모가 커지게 된다면)/네이버, 카카오 로그인(인증 방식)/figma(레이아웃 작성), stitch(디자인 다듬기)/pytest(파이썬 테스트), jest(자바스크립트 테스트)

필수 요소 : 군대 음식 파는 쇼핑몰, 군대 급식 식단 확인

부가 요소 : 모병 모집, 커뮤니티, 전역 날짜 확인, 국방 뉴스, 예비군

- 필수 요소 목록
    - 군대 음식 파는 쇼핑몰 기능
        1. 전체 음식들을 불러온 후, 리스트에 나열해서 나오게 됨
            1. 각 음식들마다 불러오는 위치는 데이터베이스(postgreSQL)에 저장
                1. 각 데이터는 이름(문자열), 가격(숫자), 칼로리(숫자), id(숫자)를 가져옴
            2. 썸네일 이미지는 storage에 저장
            3. storage, 데이터베이스에서 각 음식 별 데이터를 불러서 백엔드 부분을 거쳐서 클라이언트에 각각 표시
            4. 데이터는 이름 기준, 가나다 순으로 불러옴
            5. 비동기의 형태로 id를 1번부터 시작해서 8개 단위로 불러오며(페이지 당 표시할 음식들이 총 8개이기에, 첫 화면은 id:1~8까지) 다음 페이지 이동 시, 다음 id를 비동기 방식으로 불러옴(2페이지에서는 9~16), SELECT FROM WHERE 절을 기반으로 ID를 8단위로 나누어놓으면서 만들 수 있음
        2. 각 리스트에 있는 음식의 카드 아이콘의 버튼 클릭 시, 바로 결제창이 뜨고(TOSS 또는 네이버 결제창 도입) 결제 실행(실제 결제가 아닌 가상의 결제 형태)
            1. 로그인을 한 상태에서는 쿠폰 결제가 가능하도록 함, 쿠폰 결제는 개인적으로 쿠폰을 갖고 있어야 하기 때문
        3. 각 리스트에 있는 음식의 장바구니 아이콘의 버튼 클릭 시, 장바구니에 해당 리스트가 추가됨, 팝업
            1. 장바구니에 추가 시에, 백엔드를 거쳐 데이터베이스에 해당 음식에 관한 데이터들을 추가하게 됨
            2. 로그인을 한 상태에서는 쿠폰 결제가 가능하도록 함, 쿠폰 결제는 개인적으로 쿠폰을 갖고 있어야 하기 때문
        4. 하단의 페이지 버튼 클릭 시, 이전, 혹은 다음 페이지로 이동하게 됨(각 페이지마다 8개 씩 새로운 데이터를 받아내어 표시)
    - 군대 급식 식단 확인
        1. 현재 날짜에 대응되는 급식 데이터를 불러옴
            1. 공공데이터 json을 기반으로 각 날짜마다 급식 데이터를 가져오기(날짜: DATA 배열 내의 객체의 dates, (조/중/석)식 표기:DESCRIPTION[”brst”]/DESCRIPTION[”lunc”]/DESCRIPTION[”brst”], 메뉴 내용:DATA 배열 내의 객체의 brst/lunc/dinr) 가져온 데이터는 날짜 및 (조/중/석)식을 제목으로 표기
            2. 상단에 날짜 이동 부분에 대해서는 현재 날짜를 화면상에 표기
        2. 해당 끼니에 해당되는 아이템 내에서 상세 보기를 클릭시, 팝업 형태로 나타나면서, DATA 배열 내의 각 객체 내용들을 불러오기(날짜, 끼니의 메뉴, 개별 칼로리)
        3. 끼니 리스트의 상단의 날짜 관련 요소는 클릭 가능, 
            1. 왼쪽 화살표 클릭 시, 가운데의 날짜 글자가 이전 날짜로 표시되고, 오른쪽 화살표 클릭 시, 가은데의 날짜 글자가 다음 날짜로 표시됨, 중앙의 날짜 클릭 시, 달력이 뜨게 되고 해당 달력에서 날짜 이동 시, 해당 날짜에 해당되는 날짜가 표시됨
            2. 날짜가 변경되면, 해당 날짜에 해당되는 데이터를 찾은 후, 그 데이터를 다시 불러와서 표기함
        4. 검색창에 식단과 관련된 음식 메뉴를 검색 시, 그 음식 메뉴명에 해당하는 식단표의 날짜를 불러옴(예: 2026.06.03 날짜의 식단 데이터들을 화면 상에 불러온 상태에서 검색창에 “제육덮밥” 검색 시, 제육덮밥이 포함된 날짜 중에 가장 최근의 데이터를 불러옴, 만약에 현재가 6월 3일이고 6월 3일 범위 중에 근처 날짜들을 파악하면서 6월 1일에 제육 덮밥이 있다면, 6월 1일의 데이터를 불러옴), 데이터베이스 상에서 현재 날짜를 기점으로 앞의 날짜와 뒤의 날짜를 파악하면서 
- 부가 요소 목록
    - 모병 모집
        1. 모병 모집 관련 데이터(사진 및 정보 데이터)들을 4개를 불러옴
            1. 사진은 스토리지에 따로 저장을 하여 스토리지 ⇒ 서버 ⇒ 클라이언트로 불러오는 방식
            2. 모병 모집 정보 데이터(지원자격, 혜택, 접수 기간의 형태)는 데이터베이스에 저장이 되어 데이터베이스 ⇒ 서버 ⇒ 클라이언트로 불러오는 방식
            3. 시작은 id:1부터 4까지 UI 상에 비동기적으로 불러와서 보여줌(총 4개를 보여주어야 하기에) ⇒ 하단의 공고 페이지에서 이전/다음 버튼 클릭 시, 페이지를 이동하게 되고 해당 데이터들도 비동기의 형태로 4개의 데이터들을 불러옴, SELECT FROM WHERE를 기반으로 id를 4단위로 나누어서 불러오도록 함
            4. 검색창에 검색어를 입력을 하게 되면 해당 검색어에 맞추어 모병 모집 관련 데이터가 모두 비동기적으로 나오게 됨
            5. 모병 모집 버튼을 누르게 되면 팝업이 뜨게 되면서 지원 신청을 할 것이냐는 내용이 보여지게 됨, 신청 팝업에서 신청 버튼을 클릭하게 되면 데이터 베이스에서 해당 신청했던 내용들을 기록하게 됨. 컨텐츠들이 나오는 영역에서 모병 모집 기록을 확인하게 되면 해당 모병 모집을 했던 기록이 기록됨.

        1. 화면 상에서는 전체 게시글 데이터 중에 최근에 작성한 게시글 데이터 10개를 먼저 보여줌(id1~10)
            1. 게시글 데이터는 데이터베이스에 저장이 되며, 하단의 페이지 이동 버튼을 누를 때 마다 해당 페이지에 맞춰서 날짜에 맞추어 게시글을 비동기적으로 불러옴(2페이지에서는 최근의 10개의 게시글 이후의 데이터 10개를 가져오는 형태, id11~20과 같이)
        2. 검색어를 통해 키워드 검색 시, id, 날짜, 작성자, 제목, 내용 중에서 선택하여 비동기적으로 해당되는 것들을 모두 불러올 수 있음, 데이터베이스에서 SELECT FROM WHERE절을 기반으로 특정 조건을 만족하는 데이터를 찾아서 불러오는 형태
            1. 게시물의 데이터는 각각 id(int 형태의 숫자), 작성자(문자열), 제목(문자열), 날짜(문자열, “-”라는 부가적인 단어도 포함되어있기에), 내용(문자열)의 형태로 되어있음. 
        3. 로그인이 되지 않는다면 다른 사람들의 게시글을 볼 수는 있지만 그 외의 생성, 수정, 삭제에 대해서는 진행할 수 없음. 로그인 할 때 일반 권한에서는 자신의 게시글을 CRUD(생성, 읽기, 수정, 삭제) 모두 진행할 수 있지만, 타인의 게시글에 대해서는 건드릴 수 없고, 읽기만 가능함. 관리자 권한에서는 모든 게시글들에 대해 수정을 제외하고 나머지 작업(읽기, 생성, 삭제)이 가능함, 관리자는 자신이 작성한 게시글에 대해서는 수정은 가능
            1. 로그인 진행 시, 관리자 권한에서의 계정 데이터를 지정하기 위한 데이터베이스를 따로 지정하여 그 데이터베이스에 추가를 한 후, 로그인을 할 때 그 데이터베이스와 일치하는 계정 값이 들어온다면, 인증 권한을 더 강하게 줌, 해당 계정에서는 타인의 게시물 수정을 제외하고 모든 권한(생성, 읽기, 삭제)을 열 수 있도록 지정
            2. 일반 권한에서는 회원 가입을 통해 일반적으로 계정이 만들어지며, 해당 권한에서는 로그인을 하게 될 때 일반 권한에서는 전체 게시글들을 볼 수 있고, 게시글을 작성 가능하며, 자신의 게시글에 대해서는 수정 및 삭제가 가능, 
            3. 로그인이 안된 상태에서는 전체 게시글들에 대해서는 읽기가 가능하지만, 다른 게시물들에 대해서는 모두 생성, 수정, 삭제는 이루어지지 않음
        4. 로그인을 한 상태라면, 게시물에 대해서 댓글을 달 수 있고, 자신이 단 댓글은 자신이 직접 달 수 있고, 수정이 가능하며, 삭제도 할 수 있음, 관리자는 남의 댓글도 삭제 가능함(댓글을 다는 행위에 대해서는 게시물에 대한 권한들과 동일)
